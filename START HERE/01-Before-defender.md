 Before Microsoft Defender: Foundational SOC Training

Before building and operating my own SOC lab environment in Microsoft Defender and Sentinel, I developed a strong foundation in Security Operations concepts through hands-on training using TryHackMe.

I completed multiple SOC-focused learning paths and practical simulations, which emphasized real-world analyst workflows including alert triage, log analysis, and incident response. These modules provided structured exposure to the day-to-day responsibilities of a Security Operations Center (SOC) analyst.

A key component of this training was participating in TryHackMe’s SOC simulation environments, where I worked within a SIEM platform powered by Splunk. Within these environments, I:

Investigated simulated security alerts and incidents
Queried and analyzed logs using Splunk search processing language (SPL)
Identified indicators of compromise (IOCs) such as suspicious IPs and abnormal user behavior
Practiced correlating events across multiple data sources
Developed an understanding of attacker techniques and common detection patterns

This experience laid the groundwork for my transition into building a fully independent SOC lab, where I now apply these same principles using enterprise tools such as Microsoft Defender and Microsoft Sentinel.
