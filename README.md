# SOC-Lab-Defender
Hands-on SOC lab simulating real-world incident response using Microsoft Defender, Sentinel, and Splunk.


Introduction

This repository documents a hands-on Security Operations Center (SOC) lab designed to simulate real-world cybersecurity monitoring and incident response. The primary goal of this project is to develop practical, job-ready skills by working through realistic security incidents and applying structured investigation methodologies.

The lab environment leverages tools such as Microsoft Defender, Microsoft Sentinel, and Splunk within a virtualized infrastructure to replicate a modern enterprise security stack. Through this setup, logs are generated, ingested, and analyzed to identify suspicious activity and support incident response workflows.

 Description

This project follows a continuous, scenario-based learning approach where realistic security incidents are introduced, investigated, and documented. Incidents are modeled after real-world scenarios and implemented within a controlled and authorized lab environment, allowing for practical exposure to common attack techniques and response procedures.

Each incident is independently analyzed and documented, including:

Alert triage and investigation steps
Log analysis and correlation
Threat validation using OSINT tools
Mapping to MITRE ATT&CK techniques
Remediation and recommended actions

Completed investigations are reviewed for accuracy and completeness, reinforcing proper SOC methodologies and improving analytical skills over time.

As the lab progresses, this repository will be updated with:

Detailed incident reports
Screenshots of detection and analysis workflows
SIEM queries (Splunk / Microsoft Sentinel)
Configuration steps and troubleshooting processes
Lessons learned from each scenario

All scenarios are conducted within a private lab environment and are designed to safely replicate real-world attack patterns without impacting production systems.
